Reflection on the Validation Process

Running all four pages through the W3C Markup Validation Service surfaced a small but instructive set of issues that I resolved before taking the final screenshots submitted above.

On the site template, the validator initially flagged a missing lang attribute on the <html> element in an early draft that was shared across all pages. Adding lang="en" resolved the error immediately and improved screen-reader compatibility for all pages that inherited the template. A secondary warning about a <section> element used in the statistics strip lacking a visible heading was resolved by inserting a visually hidden <h2> using a .sr-only utility class, preserving the visual design while satisfying both the validator and WCAG 2.1 requirements.

On home.html, one warning noted that the hero <section> lacked an accessible heading. I resolved this by ensuring the <h1> was structurally inside the section rather than a sibling. I also confirmed that the CSS custom property declarations used in the embedded <style> block were valid CSS3, which the validator accepted without issue.

On gallery.html, the validator initially flagged two dynamically generated thumbnail images that were missing alt attributes, an oversight in the JavaScript template literal used to render the gallery grid. Correcting the template string to include a descriptive alt value for each image resolved both errors and improved keyboard and screen-reader accessibility at the same time.

On content_ST1.html, the validation passed cleanly with no errors. I confirmed that all internal anchor targets referenced by the sticky sidebar navigation were present in the document, and that the scroll-margin-top declarations on each <section> were valid CSS and did not affect HTML validity. The ARIA labels applied to the smooth-scrolling anchors were also verified as standards-compliant.

Overall, the validation process reinforced the value of building from a well-structured template. Because the global header, footer, and navigation were validated once at the template level, all pages that inherited that structure benefited from a clean baseline. It also highlighted how accessibility and validity are deeply intertwined, the fixes that satisfied the validator, such as adding ARIA labels and hidden headings, consistently made the pages more usable for assistive technologies too.
