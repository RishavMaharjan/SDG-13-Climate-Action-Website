User Profile Page Implementation

Technical Description

The User Profile page was developed using a pure HTML and CSS structure. Instead of JavaScript, I utilized a multi-part <form> to collect user data regarding personal details, climate interests, and sustainability goals. I utilized <fieldset> and <legend> tags to logically group the information.

I implemented a visual Progress Bar using the HTML5 <progress> element, styled with CSS to match our site's green color palette. The layout uses CSS Flexbox to ensure the form fields align perfectly on both desktop and mobile devices.

Accessibility

Every input field is explicitly associated with a <label> using the for attribute, allowing screen readers to identify fields easily. I added required attributes to essential fields to provide immediate browser-level validation, adhering to WCAG 2.1 standards.

SVG Sitemap

Technical Description

The sitemap is built using Scalable Vector Graphics (SVG). I manually coded the <rect> nodes and <path> connectors within the HTML document. To make it interactive, I wrapped the SVG shapes in anchor tags (<a>).

Accessibility

I included <title> tags inside each SVG element to provide alternative text for screen readers. Keyboard navigation was enabled by setting tabindex="0" on the nodes, following technical tutorials on SVG accessibility.

Content Page: Global Warming Impacts

Technical Description

This page utilizes a CSS Grid structure to present a clean, educational layout. I strictly followed the heading hierarchy (H1 to H3) for semantic clarity. The content was inspired by United Nations reports on Goal 13.

Accessibility

Semantic HTML5 landmarks like <article> and <aside> are used. A "Go to Top" button was created using CSS position: fixed to ensure easy navigation back to the main menu.

Challenges and Lessons Learned

SVG Coordination: Manually calculating coordinates for the sitemap branches was a technical challenge. I learned how to use SVG coordinate systems to create clean diagrams.

Validation: Resolving duplicate IDs in the SVG file was necessary to pass the W3C Validation test with no errors.
