Splash Screen Validation Report

The Splash Screen (splash.html) was validated using the W3C Markup Validation Service and passed with no errors. The page relies on a <meta http-equiv="refresh"> tag for auto-redirect, which is a valid HTML construct per the HTML5 specification. The CSS-only loader animation was verified to use only standard @keyframes syntax. The JavaScript countdown and skip button were confirmed to have no impact on markup validity. All inline styles were reviewed and removed in favour of class-based rules to ensure clean, standards-compliant output.

Action Impact Simulator Validation Report

The Action Impact Simulator (ais.html) passed W3C validation with no errors. The six action cards each include a thumbnail image with descriptive alt text, a heading, a paragraph description, and a data attribute used by JavaScript to retrieve the card's impact score, all of which are valid HTML5 constructs. Custom data-* attributes (data-points) were used instead of non-standard attributes to store impact values, which the validator accepted without warning. The embedded <style> block was reviewed to confirm it contains only valid CSS2.1/CSS3 declarations. The dynamic background-change logic is entirely JavaScript-driven and does not affect HTML validity.

Content Page Validation Report

The Content Page (content_ST2.html), covering Renewable Energy Solutions as part of the SDG 13 topic, passed W3C validation with no errors. The full heading hierarchy from <h1> through <h6> was verified to follow a logical document outline with no skipped levels. Internal anchor links and the Go to Top button were validated to reference existing id attributes on the page. All four images include descriptive alt text. The active page state in the navigation is implemented using an aria-current="page" attribute, which is both valid HTML5 and improves accessibility for screen-reader users.

Reflection on the Validation Process

Running all three pages through the W3C Markup Validation Service surfaced a small but instructive set of issues that I resolved before taking the final screenshots submitted above.

On splash.html, the validator initially flagged a stray closing </div> tag left over from an early draft of the countdown container. Removing it resolved the error immediately. A related warning noted that the <noscript> fallback message sat inside a <p> element that already contained block-level content; restructuring the noscript block as a sibling paragraph cleared the warning.

On ais.html, two of the original action card images were missing alt attributes — an oversight made during rapid prototyping. Adding meaningful alt text (e.g., "Person cycling to work instead of driving") fixed both errors and improved accessibility at the same time. Correcting these ensured the page passed the W3C validation and meets basic accessibility standards. Overall, the validation process confirmed that valid code is the foundation of an accessible and professional user experience, ensuring that our Climate Action website performs reliably across all devices.

The validation process also highlighted how accessibility and validity overlap, most of the fixes that satisfied the validator, such as adding alt text and proper heading hierarchy, consistently made the pages more usable for assistive technologies too.
