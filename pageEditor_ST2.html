Splash Screen

Technical Description

The splash screen uses a <meta http-equiv="refresh" content="4;url=home.html"> tag in the <head> to automatically redirect to home.html after four seconds, as specified in the lecture notes. The visible content is centred using display: flex on the body and contains an <h1> for the site title, a mission statement paragraph, all four group member names, a CSS loader <div>, a countdown paragraph updated by JavaScript, and a Skip Intro button.

The full-screen background image is set on the body using background-size: cover. A ::before pseudo-element with rgba(0,0,0,0.55) overlays the image to keep white text readable. The CSS loader is a <div> with border-radius: 50%. A CSS-only loading animation with a @keyframes spin rule, no GIF or video is used. The container uses display: flex to center the logo and title, and transition: opacity 1s ease-out to fade out the splash screen before hiding it via display: none in the script. The Skip Intro button calls window.location.href = 'home.html' and clearInterval() to cancel the timer immediately.

Accessibility

The page declares lang="en" on the <html> element. The background image is applied via CSS so it is ignored by screen readers as decorative. The Skip Intro <button> is a native element and is keyboard-focusable by default. The countdown paragraph uses aria-live="polite" so updates are announced to assistive technologies.

Action Impact Simulator

Technical Description

The page is structured with <header>, <main>, and <footer>. Inside <main> a <section class="card-grid"> holds six action cards and an <aside id="feedback"> displays the live score. Each card is a <div class="card"> with a data-points attribute (1–5), an <img> with alt text, an <h3> title, a description paragraph, and a points badge.

The grid uses grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)) to reflow responsively without media queries. A .card.selected CSS rule applies a coloured border and glow when JavaScript adds the class. The toggleCard() function calls classList.toggle('selected') then updateFeedback(), which sums parseInt(card.dataset.points) across all selected cards. The three impact levels are commented in the code: Each choice is a <button> that triggers a JavaScript function to update the simulator's state. The background image of the dashboard document.body.style.backgroundImage is set directly, a technique learned in the module to create dynamic theme changes. The layout uses a responsive CSS Grid to arrange the impact cards. use tabindex="0" and a keydown listener so they can be activated with Enter or Space. All images have descriptive alt text. The feedback <aside> uses aria-live="polite" so score changes are announced without interrupting other screen reader. Colour alone is not used as the sole indicator of selection, a visible border change accompanies the state transition for better accessibility.

Accessibility

Cards use tabindex="0" and a keydown listener so they can be activated with Enter or Space. All images have descriptive alt text. The feedback <aside> uses aria-live="polite" so score changes are announced without interrupting other screen reader output. Colour is never the sole indicator of selection — a visible border change accompanies every state change.

Content Page

Technical Description

The page uses a two-column CSS Grid (grid-template-columns: 220px 1fr) with a sticky <aside> side navigation alongside the main <article>. The aside uses position: sticky; top: 90px so it stays visible while scrolling. The article is split into six <section> elements with unique id attributes matching the side navigation anchors. Heading hierarchy runs strictly from <h1> down to <h6>. Images are wrapped in <figure> and <figcaption> and styled with object-fit: cover, border-radius, and box-shadow. A Go to Top link (<a href="#top">) sits at the end of each section. The active nav link is marked with class="active" and aria-current="page".

Accessibility

The <aside> carries aria-label="Page sections" so screen readers distinguish it from the main content. All images have descriptive alt text. Internal anchors allow keyboard users to jump to any section directly. The Go to Top link includes aria-label="Return to top of page". Body text maintains a contrast ratio above 4.5:1 against the white background, meeting WCAG 2.1 Level AA.

Challenges and Lessons Learned

Topic Description / Solution
Challenge 1: CSS-Only Loader The brief required an animation without GIFs or videos. I researched and implemented a CSS-only <div> with an asymmetric border, three sides transparent, one coloured, combined with a transform: rotate(45deg) and @keyframes spin. This was a fun UI challenge that didn't require any images.
Challenge 2: JavaScript Background My first instinct was to toggle a CSS class, but the brief required the change through JavaScript directly. I switched to setting document.body.style.backgroundImage, which assigns an inline style and overrides stylesheet rules, satisfying the requirement clearly.
Challenge 3: Readability White text was hard to read against lighter areas of the splash photo. I added a ::before pseudo-element on the body at rgba(0,0,0,0.55) to darken the image layer without replacing it, then verified the contrast ratio using the WebAIM Contrast Checker.
Lessons Learned Reading the brief carefully prevents wasted effort—catching the "no external libraries" requirement early saved me from having to refactor the simulator's card interactions. I also learned how to structure a single-page application using pure JavaScript without a framework like React.
